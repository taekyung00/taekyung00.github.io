Project Overview

Hello Quad is the first assignment in the CS250 Computer Graphics II series at DigiPen Institute of Technology. The objective is to build the foundational OpenGL rendering pipeline from scratch and use it to display an animated, textured quad on screen.

The project targets three platforms simultaneously — Windows (OpenGL 3.3), Web (WebGL 2 via Emscripten), and Linux (OpenGL ES 3.0) — using a shared GLSL ES 3.0 shader codebase and a cross-platform CMake build system.

Goals

  • Implement the core OpenGL abstraction layer: VertexBuffer, IndexBuffer, VertexArray, Texture.
  • Upload vertex data (position, color, UV) to the GPU and render it with a custom GLSL shader.
  • Add interactive animation driven by mouse input and real-time uniforms.
  • Practice hot-reloading shaders and textures without restarting the application.

Tasks Completed

VertexBuffer

Wraps an OpenGL VBO. Allocates GPU memory with glBufferData and supports sub-region updates via glBufferSubData. Uses RAII — the destructor calls glDeleteBuffers.

IndexBuffer

Wraps an OpenGL IBO. Stores index type (GL_UNSIGNED_INT etc.) and element count so DrawIndexed can issue the correct glDrawElements call.

VertexArray

Manages a VAO. AddVertexBuffer pre-computes the full interleaved stride, then calls glVertexAttribPointer / glVertexAttribIPointer for each attribute. Supports instanced rendering via per-attribute divisors.

Texture

Loads a PNG with stb_image and uploads it via glTexImage2D. Exposes UseForSlot(n) to bind to any texture unit, and runtime SetFiltering / SetWrapping helpers.

Vertex Shader Animation

Accepts a uAngle uniform and applies a 2D rotation matrix to vertex positions each frame, making the quad spin in response to mouse clicks.

Fragment Shader Animation

Uses sin(uAngle) to smoothly cycle the mix ratio between vertex color and texture color, producing a continuous color-blending animation tied to the rotation state.

Click-driven Physics

Each left-click adds an angular velocity impulse. The velocity decays each frame via exponential damping (exp(−1.5 · Δt)), giving a natural spin-and-stop feel that is frame-rate independent.

Hot Reload

The assets::Reloader polls shader and texture file timestamps every frame. Changes are applied instantly while the app is running — no rebuild required.

Technologies Used

  • C++20 — RAII resource management, move semantics, std::span
  • OpenGL ES 3.0 / WebGL 2 / OpenGL 3.3 — single GLSL ES 3.0 shader set for all targets
  • SDL2 — window creation, event loop, mouse input via window::input environment
  • Dear ImGui — in-app debug overlay
  • stb_image — PNG loading
  • Emscripten — WebAssembly compilation for the browser demo
  • CMake + Presets — unified build across Windows, Web, and Linux

Reflection

Challenges

The most subtle bug encountered was in VertexArray::AddVertexBuffer. My first attempt accumulated the stride inside the attribute loop, so each attribute received only a partial stride value. Position and color data were read at the wrong byte offsets, producing a corrupted quad. The fix was to compute the total stride in a separate pre-pass before setting up any attribute pointers.

On the build side, WSL's rename permission errors when extracting CMake dependencies to /mnt/c/ (NTFS) caused intermittent configure failures. Running only the web-release preset and cleaning the partial _deps_Linux directory resolved the issue.

Solutions Implemented

For the unbind convention: setup functions (LoadFromMemory, SetFiltering, SetWrapping) always unbind after modifying GL state, while render functions (UseForSlot) intentionally leave state bound for the upcoming draw call. This consistent pattern prevents accidental state leaks.

Frame-rate independent animation is achieved using window::DeltaTime and exponential decay (velocity *= exp(−damping · Δt)) rather than a fixed multiplier, ensuring the same physical behavior at any refresh rate.

Lessons Learned

  • Stride must be the total interleaved vertex size — computed before the attribute setup loop, not during it.
  • The IBO binding is part of VAO state; unbinding the IBO while the VAO is bound would silently break indexed drawing.
  • Exponential damping (exp(−k·Δt)) is the correct frame-rate-independent alternative to a per-frame multiplier.
  • Hot-reloading shaders dramatically shortens the iteration loop when tuning visual effects.
