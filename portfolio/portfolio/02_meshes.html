Project Overview

Procedural Geometric Modeling is the second assignment in the CS250 Computer Graphics II series at DigiPen Institute of Technology. The goal is to implement six procedural mesh generation algorithms — Plane, Cube, Sphere, Torus, Cylinder, and Cone — and integrate them into an interactive 3D viewer built on top of the OpenGL abstraction layer from the previous assignment.

All shapes share a single index buffer generation routine and are rendered with two shader programs: a flat-color shader for wireframe and normal-vector display, and a UV-textured shader for solid surface rendering. The viewer runs on Windows (OpenGL 3.3), Web (WebGL 2 via Emscripten), and Linux (OpenGL ES 3.0) from a shared GLSL ES 3.0 shader codebase.

Goals

  • Implement build_index_buffer() as a shared quad-to-triangle pattern reused by all shapes.
  • Generate correct vertex positions, surface normals, and UV coordinates for each of the six shapes using parametric equations.
  • Build an interactive demo with real-time stacks/slices controls, wireframe overlay, and normal-vector visualization.
  • Compile without warnings across Clang (Emscripten), GCC, and MSVC with -Werror.

Tasks Completed

Index Buffer

Implemented build_index_buffer(stacks, slices) which generates two counter-clockwise triangles per quad cell. The resulting pattern is shared by Plane, Sphere, Torus, Cylinder, and Cone without modification.

Plane

Grid of (stacks+1)×(slices+1) vertices centered at the origin. Position: (col−0.5, row−0.5, 0), normal: (0, 0, 1), UV: (col, row). Backface culling disabled to allow viewing from both sides.

Sphere

Spherical polar coordinates with radius 0.5. Latitude angle β ∈ [−π/2, π/2] over stacks, longitude α ∈ [0, 2π) over slices. Normal is position / radius — always radially outward. Degenerate triangles at poles are silently ignored by the GPU.

Torus

Parametric surface with major radius R = 0.35 and tube radius r = 0.15. Position uses a two-angle formula; normal is computed as (position − tubeCenter) / r, pointing outward from the tube axis. Supports arbitrary start/end sweep angles.

Cylinder

Barrel built with side normals (sinα, 0, cosα), plus two flat caps generated by add_cap(). Caps use separate vertices with outward normals (0, ±1, 0) and reversed winding for the bottom cap.

Cone

Tapered body where the cross-section radius scales as R·(0.5−h). Side normals are derived from the slope of the cone's 2D profile: normalize(sinα, TangentSlope, cosα). Tip vertices receive a special normal (0, 1, 0). Base cap added via add_cap().

Wireframe & Normals

convert_to_lines_pattern() converts the triangle index buffer into a line-pair index buffer for wireframe rendering. A separate normal-vector mesh is generated at build time. Both passes use the flat-color fill_3d shader.

Material & Camera System

Four materials (textured, textured-no-cull, wireframe, normals) with MVP uniform management. Camera smoothly interpolates to the selected shape using exponential easing on eyePosition and viewDistance each frame.

  • SDL2 — window creation and event loop

Reflection

Challenges

The main source of difficulty was linear algebra errors — deriving surface normals from first principles for each shape required careful geometric reasoning. The cone in particular demanded working through a 2D cross-section analysis to obtain the correct tangent slope, and index winding order for the cylinder and cone caps had to be reasoned from scratch to ensure consistent front-face orientation.

On the build side, the Emscripten toolchain enforced stricter warnings than MSVC (-Wsign-conversion, -Wimplicit-int-float-conversion, -Wdouble-promotion), causing the web-release build to fail even though the Windows debug build passed cleanly.

Solutions Implemented

For each shape's normal, I returned to the underlying geometric definition — perpendicular to the surface tangent — and re-derived the formula analytically before translating it into code. For the winding order issue, I traced through a small concrete example (4 vertices, 2 triangles) to verify CCW orientation before generalizing.

The build warnings were resolved by adding explicit static_cast calls wherever signed/unsigned or int/float implicit conversions occurred, and by changing a double constant to float to avoid the double-promotion path.

Lessons Learned

  • Parametric surface normals are best derived geometrically (perpendicular to the tangent plane) rather than approximated numerically.
  • A single shared build_index_buffer() significantly reduces duplication — all six shapes reuse it unchanged.
  • Strict Clang warnings (-Wconversion, -Wsign-conversion) expose real type mismatches that MSVC silently accepts.
  • Separating cap vertices from barrel vertices (with different normals) is essential for correct lighting on closed surfaces.
