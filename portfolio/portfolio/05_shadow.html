Project Overview

Shadow Mapping is the third assignment in the CS250 Computer Graphics II series at DigiPen Institute of Technology. The goal is to implement a classic two-pass shadow mapping pipeline: in the first pass the scene is rendered from the light's perspective into a depth texture, and in the second pass that depth texture is compared against each fragment's distance from the light to determine whether it lies in shadow.

The implementation extends the existing OpenGL abstraction layer with a FrameBuffer class and new Texture loading paths for depth and RGBA render targets. A dedicated Camera struct provides analytically derived view and inverse-view matrices used for both the eye camera and the light camera. Additional debug features include a light frustum wireframe and an on-screen depth map preview.

Goals

  • Implement a two-pass shadow mapping pipeline with a perspective light source.
  • Extend Texture with LoadAsDepthTexture() and LoadAsRGBA() for off-screen render targets.
  • Implement FrameBuffer with depth and color attachments and completeness checking.
  • Derive Camera::ViewMatrix() and Camera::ToWorldMatrix() analytically without glm::inverse().
  • Apply exponential fog and gamma correction (1 / 2.2) in the shadow shader.
  • Compile without warnings on Windows (MSVC) and Web (Emscripten) with -Werror.

Tasks Completed

Depth Texture

Texture::LoadAsDepthTexture() allocates a GPU depth texture using TexStorage2D on WebGL / OpenGL 4.2+ and TexImage2D otherwise. Sets GL_LINEAR filtering, GL_CLAMP_TO_EDGE wrapping, and GL_COMPARE_REF_TO_TEXTURE so that the texture can be bound as sampler2DShadow for hardware depth comparison.

RGBA Color Texture

Texture::LoadAsRGBA() allocates an empty RGBA8 render target by delegating to the existing LoadFromMemory() with a nullptr pixel pointer. Used as the color attachment of the shadow framebuffer to enable depth-map visualization.

FrameBuffer

FrameBuffer::LoadWithSpecification() creates an FBO, attaches the depth texture via GL_DEPTH_ATTACHMENT, optionally attaches a color texture via GL_COLOR_ATTACHMENT0, and verifies completeness with glCheckFramebufferStatus. Use(bind) binds or unbinds the FBO for rendering.

Camera Matrices

Camera::ViewMatrix() builds the world-to-camera matrix by assembling column vectors from the Right, Up, and Back basis and their dot products with Eye. Camera::ToWorldMatrix() is its analytical inverse — the transpose of the rotation block with Eye as the translation column — computed without glm::inverse().

Depth Pass

renderToDepthBuffer() binds the shadow FBO, sets the viewport to the shadow map dimensions, enables GL_POLYGON_OFFSET_FILL to reduce shadow acne, and renders the scene using the WriteDepth shader from the light camera's point of view. Front- or back-face culling can be toggled via ImGui to trade shadow acne against peter-panning.

Shadow Pass

renderToScreen() clears the default framebuffer with the fog color, binds the depth texture to slot 0 as sampler2DShadow, and renders the scene with the Shadow shader. The shader performs a projective texture lookup, applies Phong lighting scaled by the shadow factor, then applies exponential fog and gamma correction.

Light Frustum Visualization

drawLightFrustum() renders an NDC unit cube transformed by lightCamera.ToWorldMatrix() × inverse(lightProjectionMatrix), mapping clip-space corners to world space to outline the light's view volume as a wireframe box. Displayed only in View camera mode.

Depth Map Debug View

drawDepthTexture() draws a full-screen NDC quad with the ViewDepth shader, sampling the shadow FBO's color texture (written by write_depth.frag) to display a linearized grayscale depth map in a corner of the screen for debugging.

Reflection

Challenges

The primary source of difficulty was linear algebra — deriving the view and inverse-view matrices analytically, understanding how the shadow bias matrix maps NDC coordinates to texture space, and reasoning through the two-pass coordinate transformation chain from object space all the way to shadow-map UV coordinates.

Solutions Implemented

For the matrix derivations, I worked through the algebra on paper — starting from the definition of a rotation matrix and its transpose-inverse property — before translating each term into GLM column-vector form. Verifying that ViewMatrix() × ToWorldMatrix() yields the identity was a useful sanity check.

The Emscripten build warning was resolved by auditing all TexImage2D call sites for mismatched signedness and adding explicit casts where needed.

Lessons Learned

  • Shadow mapping requires careful tracking of coordinate spaces across two separate render passes; drawing the transformation chain on paper first saves debugging time.
  • The shadow bias matrix (scale + translate from [−1, 1] to [0, 1]) is easy to derive but easy to forget — omitting it produces no shadows at all.
  • GL_TEXTURE_COMPARE_MODE = GL_COMPARE_REF_TO_TEXTURE enables the GPU to perform the depth comparison automatically inside the sampler, eliminating manual comparisons in the shader.
  • Strict Emscripten warnings continue to surface type mismatches that MSVC silently accepts — testing the web build early catches these before they accumulate.
