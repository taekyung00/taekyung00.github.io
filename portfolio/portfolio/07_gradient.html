Project Overview

Gradient Noise is an assignment in the CS250 Computer Graphics II series at DigiPen Institute of Technology. The goal is to implement Improved 3D Perlin (gradient) noise entirely in GLSL and use it to generate seamless, tileable procedural textures. Unlike value noise, gradient noise places a random direction vector at each lattice point and interpolates the dot product of that gradient with the distance to the sample point, producing smoother and more organic results.

The noise is generated on the GPU: a fragment shader evaluates 3D Perlin noise for every texel and renders a 2D slice into an off-screen framebuffer backed by a single-channel floating-point (R32F) texture. This required extending the engine's texture and framebuffer abstractions to support new color formats. A fixed permutation table and twelve fixed gradient directions are hard-coded in the shader, and lattice coordinates are wrapped by the period so the texture tiles seamlessly.

The generated noise is visualized two ways: as a flat textured quad, and as a 3D terrain surface where a plane mesh is displaced in its vertex shader by sampling the noise as a height map. On top of plain gradient noise, the demo implements the classic Fractal Sum, Turbulence, Marble, and Wood patterns.

Goals

  • Implement Improved 3D Perlin gradient noise in GLSL using a hard-coded permutation table and twelve fixed gradient directions.
  • Use quintic smoothing and trilinear interpolation, and remap the [-1,1] result to [0,1].
  • Generate the noise on the GPU into an off-screen R32F framebuffer (render-to-texture).
  • Extend Texture::LoadAsFormat() to support RGBA8, RGBA32F, and R32F formats.
  • Extend FrameBuffer to attach color textures in a configurable format.
  • Implement the Fractal Sum, Turbulence, Marble, and Wood patterns and make the textures tile seamlessly.
  • Visualize the noise as both a textured quad and a displaced 3D surface.
  • Compile without warnings on Windows (MSVC) and Web (Emscripten) with -Werror.

Tasks Completed

3D Perlin Noise Core

gen_gradient_noise.frag evaluates Improved 3D Perlin noise per texel: it finds the surrounding lattice cell, computes the gradient dotted with the distance vector at all eight cube corners, and blends them with quintic trilinear interpolation to return a value in [-1,1].

Permutation Table & Hashing

A 128-entry permutation table (period 64 repeated twice) is hard-coded in the shader. Ken Perlin's nested indexing hashes the eight corners, and lattice coordinates are wrapped with & 63 so the table stays in range and the noise repeats with its period — the key to seamless tiling.

Twelve Fixed Gradients

Instead of random gradients, the improved algorithm cycles through twelve fixed directions (the edges of a cube). Each corner's hash selects one gradient via hash % 12, which is then dotted with the distance vector to give that corner's contribution.

Quintic Fade & Interpolation

The fractional cell position is reshaped by the quintic fade 6t⁵-15t⁴+10t³ before being used as interpolation weights, giving continuous first and second derivatives across cell boundaries so the noise has no visible grid artifacts.

Fractal Patterns

Building on the noise function, the shader implements Fractal Sum (summed octaves with lacunarity and gain), Turbulence (summed absolute noise), Marble (a sine of the coordinate distorted by an fBm term), and Wood (the fractional part of scaled noise).

Texture Format Support

Texture::LoadAsFormat() allocates textures in RGBA8, RGBA32F, or R32F using TexStorage2D on WebGL / OpenGL 4.2+ and TexImage2D otherwise, choosing the matching base format and component type for each.

Floating-Point Render Target

FrameBuffer was extended with an R32F color component and a writable ColorTexture() accessor, then attaches that texture as GL_COLOR_ATTACHMENT0 so the fragment shader can render the floating-point noise values straight into it.

Texture & Surface Views

simple.vert/.frag draw the noise as a grayscale textured quad, while surface.vert samples the noise with textureLod in the vertex shader to displace a plane mesh into a 3D terrain, reusing the same fragment shader for color.

Reflection

Challenges

The main challenges were reasoning carefully about coordinate spaces — from texture coordinates to lattice cells to the off-screen render target — and guaranteeing that the generated texture tiles seamlessly when repeated. Moving the noise computation onto the GPU also meant thinking in terms of a render-to-texture pipeline rather than a CPU loop, including the constraints of floating-point render targets and vertex-shader texture sampling.

Solutions Implemented

I built the noise function bottom-up — corner hashing, gradient dot products, then trilinear blending — and verified each step by first rendering only plain gradient noise before layering the fractal patterns on top. Keeping the lattice period a power of two let a single bitwise wrap guarantee seamless tiling across every octave.

For visualization I separated the work into two passes: one fragment shader bakes the noise into the framebuffer, and a second pair of shaders displays it either as a flat texture or as a vertex-displaced surface, reusing the same noise texture for both geometry and color.

Lessons Learned

  • Gradient noise is smoother than value noise because it interpolates dot products of gradients rather than raw values, forcing the noise to zero at every lattice point.
  • Seamless tiling falls out naturally once lattice coordinates are wrapped by the noise period — a power-of-two period keeps this true through all fractal octaves.
  • Generating noise on the GPU is dramatically faster than on the CPU, but requires care with floating-point render targets and explicit-LOD texture sampling in vertex shaders.
  • The same noise field can drive completely different looks — a flat texture or a 3D terrain — just by how it is sampled and applied.
