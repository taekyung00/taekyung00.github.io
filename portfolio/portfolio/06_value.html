Project Overview

Value Noise is an assignment in the CS250 Computer Graphics II series at DigiPen Institute of Technology. The goal is to implement a procedural value noise generator from scratch: a lattice of pseudo-random values is sampled at arbitrary coordinates and smoothly interpolated to produce continuous, tileable noise. The result is generated on the CPU into a texture that is then displayed on a full-screen quad.

The core is a templated ValueNoise<T> class that supports 1D, 2D, and 3D evaluation. Each integer lattice point is mapped to a random value through a PermutationHash, and neighboring values are blended using selectable smoothing functions (Linear, Cosine, Smoothstep, Quintic). On top of plain value noise, the demo layers several classic procedural patterns — Fractal Sum, Turbulence, Marble, and Wood — built by summing multiple octaves of noise.

Because generating a high-resolution texture is expensive, the demo distributes the work across a JobSystem of worker threads on native builds, and falls back to time-sliced chunked generation on single-threaded web builds, keeping the interface responsive while the texture updates.

Goals

  • Implement a templated ValueNoise<T> supporting Evaluate(x), Evaluate(x, y), and Evaluate(x, y, z).
  • Map lattice coordinates to random values with a wrap-around PermutationHash.
  • Implement linear, bilinear, and trilinear interpolation with selectable smoothing (Linear, Cosine, Smoothstep, Quintic).
  • Build fractal patterns — Fractal Sum (fBm), Turbulence, Marble, and Wood — by summing octaves.
  • Generate the noise texture asynchronously via a JobSystem with a chunked fallback for the web.
  • Compile without warnings on Windows (MSVC) and Web (Emscripten) with -Werror.

Tasks Completed

Value Noise Core

ValueNoise<T> stores a std::vector<T> of random lattice values sized to the noise period. Evaluate() is overloaded for 1D, 2D, and 3D input, gathering the surrounding lattice values and blending them to return a smooth value at any fractional coordinate.

Noise Coordinate

make_noise_coord() splits a float input into a NoiseCoordinate holding the integer base cell, its next neighbor, and the fractional interpolant in [0, 1] used to mix between the two lattice values.

Permutation Hash

PermutationHash converts one, two, or three integer lattice coordinates into an index into the random value array. It wraps coordinates by the noise period so the resulting noise tiles seamlessly across that period.

Interpolation & Smoothing

linear_mix, bilinear_mix, and trilinear_mix blend 2, 4, and 8 lattice values respectively. fade() reshapes the raw interpolant with the selected SmoothMethod — Linear, Cosine, Smoothstep, or Quintic — to control the visual smoothness.

Fractal Sum & Turbulence

Fractal Sum (fBm) accumulates several octaves of noise, scaling frequency by lacunarity and amplitude by gain each layer. Turbulence sums the absolute value of signed noise per octave to produce sharper, billowy detail.

Marble & Wood

Marble feeds an fBm term into a sine wave of the coordinate to produce veined bands, while Wood scales a single noise sample and keeps its fractional part to create concentric grain rings — both classic procedural texture effects.

Procedural Texture Pipeline

A Generation state machine (Setup → Working → UploadNewTexture → Done) fills an RGBA color buffer, then uploads it with Texture::UploadAsRGBA(). The texture is drawn on a full-screen quad with an aspect-correct orthographic projection and adjustable tiling scale.

Multithreaded Generation

On native builds JobSystem::DoJobs() splits the per-pixel work across worker threads. When threads are unavailable (util::CanUseThreads == false on the web), the demo falls back to a time-sliced loop that generates the texture in chunks across frames.

Reflection

Challenges

The main conceptual challenge was reasoning about interpolation across multiple dimensions — extending the same idea from blending two values in 1D to four in 2D and eight in 3D — and keeping the lattice-coordinate hashing consistent so that the noise stays continuous and tileable. Balancing texture generation cost against interactivity also required care about how work is scheduled per frame.

Solutions Implemented

I built the interpolation bottom-up, expressing the 2D and 3D cases as nested applications of the 1D mix so each level reused the one below it. Working through the octave-summation math for the fractal patterns on paper clarified how lacunarity and gain shape the final look before tuning them in the demo.

For performance I separated generation into a small state machine so the same code path drives both the multithreaded job-based route and the single-threaded chunked fallback, with the choice made at compile time via if constexpr.

Lessons Learned

  • Higher-dimensional interpolation is just the 1D blend applied recursively — seeing that structure made the 2D and 3D code straightforward.
  • The choice of smoothing function (especially Quintic) has a large visual impact on how organic the noise looks, despite being a small change in the math.
  • Fractal patterns emerge from very simple rules: summing octaves with decreasing amplitude and increasing frequency produces convincing natural texture.
  • Designing generation as a state machine made it easy to support both threaded and single-threaded targets without duplicating the noise logic.
